Test coverage + Refactoring

Unit testing

"Unit testing is a level of software testing where individual units/ components of a software are tested. The purpose is to validate that each unit of the software performs as designed."
http://softwaretestingfundamentals.com/unit-testing/

Unit testing is a piece of code that you write in order to test another piece of code.

- Ricardo Linck.

So what do we need to get started?

Infrastructure

Most languages have frameworks to create and run unit tests.

You need to learn the specifics of the framework you are going to use.

Unit test structure

Usually the system you are going to test is called System under test (sut).

  1. Arrange - Initialises objects and set values for the test
  2. Act - invokes the sut method with arranged parameters
  3. Assert - verifies that the results from the action with arranged parameters are the expected ones

A good unit test should be:

  • Small
  • Fast
  • Independent
  • Repeatable

Pros

  • Feedback loop is really small
  • Easy to spot where things are broken and side effects on changes
  • Easy to run
  • Builds confidence when adding new features
  • Builds confidence when refactoring
  • Reduced debbuging time (0 time in some cases)
  • Reduce the time of manually tests

Cons

  • Takes time to get used to them
  • Takes time to write them (not much though)
  • Bad unit tests is worse than no unit tests
  • You need to have a good design in order to unit test it (SRP at least)

More insights

  • You won't find bugs in all unit tests
  • Definitely the ones you find you'll thank finding them :)
  • Even if you don't find direct bugs, it will help with the confidence when refactoring

Types of Tests

Testing pyramid

Unit Test

Test a unit of the app without its external dependencies

Integration Test

Test the app with its external dependencies

End-to-end Test

Test the app through the UI

Sample Class

public class NameChanged
{
  public Guid EventId { get; }
  public string Name { get; }

  public NameChanged(Guid eventId, string name)
  {
    if (string.IsNullOrWhiteSpace(name))
    {
	    throw new ArgumentException("Name can't be empty.",
	      nameof(name));
    }

    EventId = eventId;
    Name = name;
  }
}

Sample Test Class

public class NameChangedTests 
{
  [Test]
  public void NullNameThrowsArgumentException() {
    // Arrange & Act & Assert
	Assert.That(() => new NameChanged(Guid.NewGuid(), null),
	 Throws.ArgumentException);
  }
  
  [Test]
  public void NotNullNameDoesNotThrowArgumentException() {
    // Arrange 
    var id = Guid.NewGuid();
    var name = "Any Name";
  
    // Act
    var sut = new NameChanged(id, name);
    var sutName = sut.Name;
  
    // Assert
    Assert.AreEqual(id, sut.EventId);
    Assert.AreEqual(name,sut.Name);
  }
}
